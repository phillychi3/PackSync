# Project Context

## Purpose

PaskSync 是一個方便旅行的管理系統

- 行程管理，規劃並排續行程，在前端介面顯示景點資訊與預定前往等資訊，並且有地圖與時間表介面可以清楚的看到如何行程規劃
- 在應用中管理和追蹤共同開支，自動計算每個人應付金額和欠款，支援多種分攤方式(平均分攤、自訂份數、按比例)，透過 Web 前端查看詳細帳單和統計資料，智能結算,提供最優化的轉帳方案
- 行李確認，共同規劃代辦事項與需要帶的物品
- 重要物品確認，每個行程都可確認是否錢包護照等重要物品位置
- 通知功能，發出提醒來確認下一個行程與檢查重要物品
- 旅行agent，透過詢問問題來快速確認行程與地點的問題

- **TypeScript/Svelte**:
  - 使用 2 空格縮排
  - 變數和函數使用 camelCase
  - 組件使用 PascalCase
  - 檔案名稱使用 kebab-case
  - **禁止使用 `<style>` 標籤,統一使用 TailwindCSS**

- **命名規範**:
  - 無註解
  - API 端點使用英文
  - Git commit 訊息使用英文

- **Frontend**:
  - SvelteKit 的檔案路由系統
  - 組件化設計
  - Store 管理全域狀態
  - 響應式設計 (RWD)

## Domain Context

### 旅行團體 (Travel Group)

- **旅行 (Trip)**: 一次完整的旅行活動,為所有功能的頂層容器,包含行程、帳單、行李清單等
- **成員 (Member)**: 參與該次旅行的使用者,每位成員皆有角色(例如:行程發起人、一般成員)
- **旅行狀態 (Trip Status)**: 籌備中 / 進行中 / 已結束

### 行程管理核心概念 (Itinerary)

- **行程 (Itinerary)**: 整趟旅行的時間序列計畫
- **景點/地點 (Place)**: 行程中的單一目的地,包含名稱、地址、座標、開放時間等資訊
- **行程項目 (Schedule Item)**: 某一天某個時段前往特定地點的安排,含預計到達/離開時間
- **日程 (Day Plan)**: 以「天」為單位彙整的行程檢視,可在地圖 (Map View) 與時間表 (Timeline View) 兩種介面呈現
- **地圖檢視 (Map View)**: 在地圖上標示當天所有地點與路線
- **時間表檢視 (Timeline View)**: 以時間軸呈現當天所有行程項目與間隔時間

### 分帳系統核心概念 (Expense Splitting)

- **帳單 (Bill)**: 一次共同消費的記錄,包含多個項目和參與者
- **項目 (Item)**: 帳單中的單一消費項目(例如:一道菜、一杯飲料)
- **付款人 (Payer)**: 實際付款的成員,一筆帳單可有多位付款人
- **參與者 (Participant)**: 參與該次消費、需分擔費用的成員
- **分攤方式 (Split Method)**:
  - 平均分攤 (Equal): 所有人平分總金額
  - 按比例 (Percentage): 直接指定每人負擔的百分比(例如: 50%/30%/20%)
  - 自訂金額 (Fixed Amount): 直接指定每人應付的固定金額
- **結算 (Settlement)**: 計算誰欠誰多少錢,透過最小轉帳次數的演算法提供最優轉帳方案
- **帳單統計 (Statistics)**: 依類別、成員、時間段等維度呈現消費分析

### 行李與物品管理 (Packing & Items)

- **打包清單 (Packing List)**: 整趟旅行需要攜帶物品的共同清單
- **物品 (Item)**: 清單中的單一待帶物品,含名稱、類別、數量
- **類別 (Category)**: 物品的分類(例如:衣物、盥洗用品、證件、藥品)
- **確認狀態 (Check Status)**: 物品是否已放入行李(未確認 / 已確認)
- **代辦事項 (To-Do)**: 出發前需完成的任務清單,可指派給特定成員

### 重要物品確認 (Critical Items)

- **重要物品 (Critical Item)**: 旅行中不可遺漏的關鍵物品(例如:護照、機票、錢包、藥物)
- **出發前清單 (Pre-departure Checklist)**: 每次搭乘交通工具或離開住所前須確認的物品清單
- **確認動作 (Confirmation)**: 成員主動確認已攜帶某重要物品的行為,附時間戳記

### 通知系統 (Notifications)

- **提醒類型 (Notification Type)**:
  - 行程提醒: 下一個景點/交通工具出發前的推播
  - 物品確認提醒: 提醒成員確認重要物品
  - 帳單提醒: 新增帳單或結算完成時通知相關成員
  - 代辦到期提醒: 代辦事項截止日前通知
- **推播通知 (Push Notification)**: 透過 PWA Web Push 發送,支援背景通知
- **通知偏好 (Notification Preference)**: 每位成員可自訂要接收哪些類型的通知

### 旅行 AI Agent (Travel Agent)

- **對話式查詢 (Conversational Query)**: 使用者可用自然語言詢問行程、地點、費用等問題
- **行程建議 (Itinerary Suggestion)**: Agent 根據目的地與偏好推薦景點與路線
- **地點資訊查詢 (Place Info Lookup)**: 即時查詢地點的開放時間、評價、交通方式
- **費用試算 (Cost Estimation)**: Agent 協助預估旅遊預算與分攤金額
